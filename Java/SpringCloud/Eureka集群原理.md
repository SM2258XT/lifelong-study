微服务RPC远程服务调用最核心的是什么？高可用，即使有几台服务器出现故障，也不影响对用户来说的使用影响。
潜在问题：试想你的注册中心只有一个，万一它出故障了，会导致整个微服务环境全都不可用！
解决办法：搭建Eureka注册中心集群，实现负载均衡+故障容错。
集群原理：注册中心互相注册，相互守望。任意一个注册中心中要有所有其他注册中心的注册实例。
